# Why I Build on Next.js Instead of WordPress (And What That Means for Your Site Speed)

{/* Matt: This is a differentiation post. The audience is a business owner comparing you to an agency that builds on WordPress (like Plexamedia). They don't care about the tech — they care about what the tech does for them. Keep it in plain language throughout. */}

---

Most web designers build on WordPress. About 40% of the entire internet runs on it. So when I tell people I don't use WordPress, the first question is always the same:

**"Why not?"**

The short answer: your website will be faster, more secure, and rank better on Google without it.

Here's the longer answer.

## What WordPress Actually Is

WordPress started as a blogging platform in 2003. It wasn't designed to be a business website builder — but because it was free and flexible, web designers started using it for everything. Over the years, people built thousands of plugins to bolt on features WordPress wasn't built for: contact forms, SEO tools, image galleries, booking systems, security, speed optimization.

That's the core issue. **WordPress needs plugins to do what modern frameworks do out of the box.**

A typical WordPress business site runs 20-40 plugins. Each one is a piece of code written by a different developer, updated on a different schedule, and tested against a different version of WordPress. They don't always play nicely together.

## The Plugin Problem

Here's what happens with plugins in the real world:

**They break each other.** You install a contact form plugin. It works great. Then your SEO plugin updates and suddenly the form doesn't submit. You didn't change anything — two plugins just stopped getting along. Now your site is silently losing leads until someone notices.

**They slow your site down.** Every plugin loads its own CSS and JavaScript files — even on pages that don't use that plugin. A "lightweight" WordPress site with 20 plugins might load 30-40 extra files on every page. Your visitor's phone doesn't care why it's slow. They just leave.

**They create security holes.** In 2023 alone, over 4,000 WordPress plugin vulnerabilities were reported. When a plugin hasn't been updated in six months, it's not just stale — it's a door left unlocked. And your web designer can't monitor all 30 plugins for security patches every week unless you're paying them to.

**They need constant babysitting.** WordPress core updates. Plugin updates. Theme updates. PHP version updates. Each update can break something. Agencies that manage WordPress sites spend a huge chunk of their time just keeping sites from falling apart — and they pass that cost on to you.

## What I Use Instead

I build on Next.js — a modern web framework built by Vercel, the same company that hosts sites for Nike, McDonald's, Notion, and The Washington Post.

But you don't need to remember that name. Here's what it means for you:

### Your Site Loads Fast — Really Fast

A Next.js site doesn't query a database every time someone visits. The pages are pre-built and served instantly, like handing someone a printed flyer instead of making them wait while you type it up.

**What you experience:** Your site loads in 1-2 seconds on a phone. A WordPress site with plugins typically loads in 3-6 seconds. Google has said that 53% of mobile visitors leave a site that takes longer than 3 seconds to load.

### No Plugins to Break

Everything your site needs is built into the code from the start. The contact form, the SEO, the image optimization, the speed — it's all part of one codebase, written to work together. There's no plugin from 2019 waiting to conflict with an update from 2026.

**What you experience:** Your site doesn't randomly break on a Tuesday because two plugins stopped getting along. No emergency "my form isn't working" calls. No "my site looks weird on my phone now."

### Google Rewards Speed

Google's Core Web Vitals — the speed and performance metrics they use to rank websites — favor fast, clean sites. A Next.js site scores high on these metrics by default. A WordPress site needs a caching plugin, an image optimization plugin, a minification plugin, and a lazy-loading plugin just to get close to the same score.

**What you experience:** Better Google rankings without paying extra for "speed optimization." The speed is built in, not bolted on.

### Security Without the Worry

No database to hack. No admin login page for bots to attack. No plugin vulnerabilities to patch. A Next.js site has a dramatically smaller attack surface than WordPress.

**What you experience:** You never get an email saying "your site has been compromised" or "we detected suspicious login attempts."

## "But WordPress Is Cheaper"

It can be — upfront. A WordPress template site might cost $500. But here's what the real cost looks like over time:

| | WordPress Template | Custom Next.js Build |
|---|---|---|
| **Build cost** | $500-$1,500 | $795-$1,195 |
| **Monthly hosting** | $10-$30 (shared hosting) | Included in care plan |
| **Plugin updates** | You or your designer, monthly | None needed |
| **Security monitoring** | $10-$30/mo (plugin or service) | Built in |
| **Speed optimization** | $50-$200 (one-time plugin setup) | Built in |
| **"Something broke" fixes** | $75-$150/hr when it happens | Doesn't happen |
| **Redesign in 2-3 years** | Yes — themes age out, plugins get abandoned | No — code doesn't depend on third-party themes |

The cheapest website is the one that works for years without surprise costs.

## "But Everyone Uses WordPress"

They do. And for some use cases, that's fine. If you're a blogger or running a massive e-commerce operation with 10,000 products, WordPress (with WooCommerce) might make sense.

But you're a local service business. You need 3-7 pages that load fast, rank well, and get the phone to ring. You don't need a content management system designed for a publishing platform. You need a site that works.

## "Can I Edit My Own Site?"

Yes. I set up a simple editing workflow for content changes — updating your hours, adding a new service, changing a phone number. For anything beyond that, your care plan covers it.

But honestly? Most business owners don't want to edit their own site. They want someone to do it right so they can focus on their business. That's why care plans exist.

## What This Means for You

You don't need to understand Next.js. You don't need to know what server-side rendering means. You need to know three things:

1. **Your site will be fast.** Faster than your competitors. Fast enough that Google notices.
2. **Your site won't break.** No plugins fighting each other. No mystery crashes.
3. **Your site will rank.** Clean code, fast load times, proper SEO structure — all built in from day one.

The tech is my job. The results are yours.
